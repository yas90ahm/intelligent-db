## 1. MISSION / THESIS / CORE CLAIMS TO PRESERVE

**Mission.** Intelligent DB is a memory substrate for AI agents that counters the two agent failure modes: **forgetting** (stateless context loss within and across sessions) and **hallucination** (inventing facts, unable to tell recall from invention). It is the deliberate inversion of a vector DB: facts are latent and surface only via **spreading activation** along structural threads (memory-palace / spider-web), never by fuzzy nearest-neighbor query.

**Settled architecture (do NOT relitigate).** Latent memory activated by traversal not query; web of webs bridged by rare cross-web threads; threads connect only on shared-entity (mechanical) + confirmed-link (the relationship); fact states `LIVE` / `PROVISIONAL` / `DEMOTED` (contradiction demotes, never deletes); provenance first-class on every strand; observed-vs-derived — "wall with a window," a derived fact may be believed and spoken but is **never its own witness**.

**The two governing invariants (every fix is checked against both).**
1. **The model is never its own witness.** It files and speaks memories; it never confirms them. No provenance → no voice.
2. **The web is never its own witness about source identity.** Trust in WHAT a fact is comes from inside the web; trust in WHO sourced it comes from OUTSIDE — the Source-Identity Layer.

**The hard theorem (the boundary no fix may pretend to cross).** Under "identity is priced, not prevented" (a patient attacker can pay a finite cost to mint independent-looking sources), there is NO purely internal rule that both (a) lets one true witness overturn a planted false canonical AND (b) stops two fakes overturning a true incumbent. The three attacks with no internal repair — contradiction-set bomb, first-arrival trap, no-second-independent-lock — are why identity is witnessed from outside. The second independent lock is **anchor-set disjointness (MIS)**, sourced from the external anchor layer.

**The unifying design rule (load-bearing, applies to every incumbency-touching fix).** Incumbency may be rewarded by independent-corroboration **DEPTH** (the MIS-priced quantity), and **NEVER** by reputation **MAGNITUDE**, **AGE**, or **ARRIVAL ORDER** — those three are exactly what the impossibility theorem proved an attacker controls. Every incumbency-touching gate keys on depth, enforced as a code-level assertion (§5, OD-3 asserts).

**The measured CORE CLAIMS that must survive every fix (PAPER.md §6, the acceptance band):**
- **Recall p50 flat ~1.93 → 2.08 ms across 1k → 1M facts** (recall is O(local web), not O(total); lit-set ~77). Band: **≤ 2.60 ms @1M.**
- **Writes:** ~25 µs p50 single-fact; `writeFactsBatch` ~98.9k–104k facts/s. Band: **writeFact p50 ≤ 125 µs; batch ≥ 80k/s.**
- **24/24 cheap-Sybil resistance** (structural MIS collapse + co-asserter-tie DEFER), AND the paired **honesty-control** (a genuinely-paid distinct-anchor fleet DOES overturn — "priced, not prevented"). Zero tolerance in both directions.
- **Audit proofs O(log n)** (RFC-6962 inclusion / consistency).
- **Contradiction-integrity end-task** (20 scenarios; 20/20 true-LIVE + 100/100 Sybil-DEMOTED pre-LLM asserts).
- Structural guarantees: demote-never-delete / immortal archive stub; bare-key rep cap ~0.05; MIS exactness + `min(distinctClassCount, maxSetSize)` clamp; monotone-non-increasing walk termination.

### 3.0 The load-bearing primitive: ONE agreement-set root-union `R`, behind the engine-owned-evidence invariant

Three of the breaches (RC-1 amortization, RC-2 class-vs-root, RC-3 corroborator-naming) are three consumers of the SAME missing quantity. Implement it ONCE and seal the injection surface that let the red-team harness "defend" while the documented contract "breached."

#### The engine-owned-evidence invariant (OD-8) — BINDING

> **No security gate consumes a caller-supplied EVIDENCE proxy. The engine CONSTRUCTS every gate input from the trust layer it owns (identity + reputation + the agreement index); from the caller it accepts only an INTENT flag.**

This is the root-cause framing for the largest class of breaches: patching callback *defaults* leaves the gate injectable, which is exactly why the harness defended while the contract breached. We change the **signatures** so injection is impossible, at three call sites:

1. `api.ts:416-419` — `AdjudicateOptions.highImpact: HighImpactContext` → `highImpact?: boolean` (INTENT only).
2. `api.ts:202-216` — `RatifyInput.corroboratingStrandIds` REMOVED (or retained deprecated/ignored).
3. `consolidation.ts:604-620` — `HighImpactContext` stays in the pure module UNCHANGED; only its *construction* moves into the engine. The single path from the public API to this parameter is the boolean → `#buildHighImpactContext` bridge at `api.ts:836`.

New engine method `#buildHighImpactContext(members): HighImpactContext` (`api.ts:836`) builds the three callbacks from `#reputation` and `#identity`. Requires one additive field `ReputationState.lastContradictionAt: EpochMs | null` (set in the `contradict`/`disownSweep` paths at `reputation.ts ~351/611` and the disown crater); fallback if absent = fail-closed (any source with `contradictedCount > 0` and unknown timestamp ⇒ recency-clean window fails ⇒ DEFER). The pure-consolidation ↔ ledger ↔ engine layering is preserved.

#### The R-primitive (OD-6) — `R(entity, attribute, value)`

> `#R(target)` := `identity.independentRootCount(` root-UNION over the target's provenance + every agreeing LIVE strand's roots, deduped `)` — the number of mutually anchor-independent ACTORS backing the WINNING value.

- **[STRUCTURAL]**, pure reuse of the existing Bron–Kerbosch MIS (`identity/index.ts:262-497`), MIS exactness + `min(distinctClassCount, maxSetSize)` clamp inherited verbatim, zero new algorithm.
- **Exact seam:** new engine helper `#R(target: Strand): number` (closes over `#deriveAgreementSet`); per-VALUE (requires same `content_hash`, the mechanical value fingerprint). Replaces the single-strand `winner.provenance` reading at `consolidation.ts:619` (`anchorClassCountOf`) and `api.ts:965`. **Correction the literal F1 wording must absorb:** the input is the agreement-set root union across all LIVE strands asserting the value, NOT the literal `winner.provenance` — corroboration arrives both as `ratify` appending an external root AND as separate agreeing strands; the single-strand reading under-counts the latter and OVER-DEFERS legitimately-corroborated facts (the "fp-1 CLOSED" is a harness artifact).
- **Feeds all three:** F1 (`anchorClassCountOf := #R`), F3 (same `#deriveAgreementSet` basis → no drift), F4a (defaulted `agreementRootCountOf` callback, engine passes `#R`). Three ad-hoc agreement sets would drift and the seam between them becomes the next breach.
- **Perf:** O(co-asserters) index-scoped lookup (`strandsByEntity`/`strandsByAttribute`), at adjudicate/ratify/disown time — OFF the recall and batch-write hot paths. VETO honored: MIS never touches the activation walk.

## 5. VALIDATION & ACCEPTANCE GATES

**The one methodological change (non-negotiable).** "Breach count down" is the WRONG single metric — most fixes convert BREACHED → DEFERRED, which is integrity-safe but an availability/DoS liability. Score every attack on TWO orthogonal axes:

| Label | Integrity | Availability |
|---|---|---|
| DEFENDED | false value rejected | resolved, no human |
| SAFE-DEFER | no false value won | one bounded, dedup-able enqueue |
| DOS-DEFER | no false value won | unbounded / amplified enqueue or bridge-starve |
| BREACHED | **false value LIVE** | — |

A fix converting BREACHED → DOS-DEFER is NOT a pass (it traded an integrity breach for an availability breach).

**Regression suites + bars.**
- **Red-team (108): per-id MONOTONE-SAFETY gate.** An id may only move along `BREACHED → SAFE-DEFER → DEFENDED` (and `DOS-DEFER → SAFE-DEFER → DEFENDED`); any wrong-direction move FAILS the build. Pre-register the 59 breaches into **Bucket A** (close → DEFENDED, ~17), **Bucket B** (convert → SAFE-DEFER, ~12), **Bucket C** (frozen documented residual, ~30, each carrying a machine-checked `residual_class` tag: `priced-not-prevented` | `offline-class-assignment` | `operational-witness-sinks`). **Headline target: 59 → ≤ |Bucket C| (≈28–30), zero integrity breaches outside the frozen set.**
- **Cross-DB 24/24:** poison-correct == 24/24, zero tolerance, AND the paired honesty-control ("paid fleet DOES flip") must still pass — both as asserts in one test (guards over-fixing, preserves priced-not-prevented).
- **Deployment perf:** recall p50 ≤ 2.60 ms @1M; writeFact p50 ≤ 125 µs; batch ≥ 80k/s; audit proofs O(log n) across the 1k→1M sweep. **G5 is BLOCKING for A1 specifically.**
- **QA / contradiction integrity:** pre-LLM asserts 20/20 true-LIVE + 100/100 Sybil-DEMOTED stay green; end-task qwen ≥ 0.90, llama ≥ 0.95 (baseline − 0.05 reader-noise band); raw-poisoned arm stays 0.00–0.15; LoCoMo recall@10/MRR within ±0.01 (tripwire).
