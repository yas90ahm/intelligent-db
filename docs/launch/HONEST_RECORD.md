# The Honest Record — every disclosed defect, who caught it, and what changed

This project's security thesis is "priced, not prevented" — and the same standard is
applied to its own defects. Bugs are not quietly fixed and forgotten; each one is recorded
with who caught it, what changed, and the regression test that would catch its return. The
table below is that record, in rough chronological order, with every row sourced from the
document that first disclosed it.

| What could have gone wrong | Who caught it | What changed | Where it's proven |
|---|---|---|---|
| **Fresh-SSO pricing hole**: the initial crypto-free design would have given a brand-new SSO tenant (five minutes, free) the same trust weight as a verified organization — quietly reopening the cheap-identity attack the layer exists to stop. | Adversarial design review, before any code was written. | Fixed in the design: bare tenant membership is priced near email-grade; organization-grade weight requires a registry-config-verified custom domain. | [`REBUILD_SUMMARY.md`](./REBUILD_SUMMARY.md) §3; the tenant-domain rule ships in `identity/trustRegistry.ts` (see `CLAUDE.md` pillar 1) |
| **Asserted-attribution regression**: removing signatures means the audit trail's record of *who* wrote something is asserted, not cryptographically proven — an insider with write access can rewrite local history and re-verify green. | Disclosed bluntly by the design review itself, at design time — not discovered later. | Accepted as the rebuild's named headline trade-off, with two executable mitigations: exported `chainHead()` checkpoints a rewrite cannot reproduce, and real-time `AppendSink` shipping (ship-before-write) so a rewrite diverges from the already-shipped copy at the forged record. | [`REBUILD_SUMMARY.md`](./REBUILD_SUMMARY.md) §3, §5; executable in `cryptoFreeTrust.test.ts` and `auditShipping.test.ts` |
| **Relay-fix hole 1 — contradiction laundering**: a fact that *contradicted* a cited source could still inherit that source's identity class, collapsing a genuine independent dispute into a fake same-class echo. | Relay-fix adversarial review (Phase 1), before landing. | The echo gate: class inheritance now requires actually *agreeing* with the cited witness (same content hash + attribute); a refused witness keeps its citation but contributes no class. | [`REBUILD_SUMMARY.md`](./REBUILD_SUMMARY.md) §3; regression in `src/__tests__/relayFix.test.ts` (see `CLAUDE.md`, relay fix) |
| **Relay-fix hole 2 — punishing the honest upstream**: disowning a fraudulent middleman could claw back reputation from the honest source *upstream* of him. | Relay-fix adversarial review (Phase 1), before landing. | Inherited classes are marked (`inheritedClass`) and excluded from the disown sweep's tainted-class set — the clawback stops at what the fraudster actually owned. | [`REBUILD_SUMMARY.md`](./REBUILD_SUMMARY.md) §3; the disown-a-relayer taint-bound case in `src/__tests__/relayFix.test.ts` |
| **`ratify()` was not atomic**: the one belief-changing verb not wrapped in the shared transaction helper — a crash mid-ratify could promote a strand with no matching reputation credit. | External launch code review (`CODE_REVIEW_IDENTITY_RATIFICATION.md`), after the project had already declared "CRITICAL: none remain." | `ratify()` now delegates to a private `#ratifyImpl` inside the same `withTxn` helper as the other four compound ops. | [`BUGFIX_REPORT.md`](./BUGFIX_REPORT.md) Bug A; regression test in `src/__tests__/atomicCompound.test.ts` (forced mid-ratify throw ⇒ full rollback) |
| **`independentSources()` failed open**: an unregistered source read as *independent*, letting a never-registered "approver" pass the dispute-approval independence gate for free. | External launch code review (`CODE_REVIEW_IDENTITY_RATIFICATION.md`). | The unregistered-side branch flipped from fail-open to fail-closed: an anchorless source can never be independent of anything. | [`BUGFIX_REPORT.md`](./BUGFIX_REPORT.md) Bug B; regression test in `src/ratification/engineAdjudicate.test.ts` (unregistered approver rejected) |
| **`description_value` hardcoded to 0**: the `LOW_UNIQUE_VALUE` below-COLD eviction gate was dead weight — it permitted eviction regardless of a strand's actual uniqueness. | External launch code review (`CODE_REVIEW_TRAVERSAL_FORGETTING_STORE.md`). | A real, zero-dependency order-0 Shannon-entropy estimate is now computed over the fact's payload at write time (documented as a heuristic with a named residual, not oversold). | [`BUGFIX_REPORT.md`](./BUGFIX_REPORT.md) Bug C; two regression tests in `src/__tests__/smoke.test.ts` (unique payload denied eviction; duplicate payload permitted) |
| **MCP boundary softness**: recalled facts crossed the tool boundary without their belief-state labels, and untrusted fact text could forge the structure of a reviewer's approval message. | Phase-4 adversarial review of the dispute horn. | Recall rendering now labels non-LIVE states (`[DEMOTED]` / `[PROVISIONAL]`), and both renderers escape control characters + quote-delimit untrusted payload text. | [`REBUILD_SUMMARY.md`](./REBUILD_SUMMARY.md) §3; injection-resistance cases in `src/__tests__/disputeHorn.test.ts` (see `CLAUDE.md`, Phase 4) |
| **Two-step-relay review-queue gap**: the consulted-but-uncited influence review queue looked exactly one hop deep, so an A→c1→b1 relay escaped human review — a standing red-team breach. | Phase-5 red-team triage. | The queue is now a transitive backward BFS over consulted-strand edges (cycle-safe, deterministic — and still human-review-only; it never auto-punishes). BREACHED → DEFENDED. | [`REBUILD_SUMMARY.md`](./REBUILD_SUMMARY.md) §3; [`docs/ARCHITECTURE_BENCHMARKS.md`](../ARCHITECTURE_BENCHMARKS.md) §10.2 Bucket B; regression in `src/ratification/disownHardening.test.ts` |
| **Subdomain-seam bookkeeping error**: a red-team attack kept scoring as a breach because the *test harness* modeled a no-PSL system that no longer exists — the shipped code already collapses subdomains to eTLD+1. | Phase-5 independent external audit — of the benchmark, not the code. | The spec was corrected to attack the real shipped Public Suffix List resolver (so it regresses to BREACHED if the PSL is ever unwired). ## Why keep this table

Three of these rows were found in code this project's own documentation had already
described as "adversarially verified." CLAUDE.md's gap list states the lesson verbatim:

> Worth stating plainly: these three were real, exploitable gaps in code this document
> previously described as "adversarially verified" and "CRITICAL: none remain." That claim
> should be read as *strong, not exhaustive* — a second independent review found what the
> first one's own test suite didn't cover. Nothing here changes the "none remain" CRITICAL
> verdict going forward (all three are now fixed and tested), but the discovery process
> itself is part of the honest record.
