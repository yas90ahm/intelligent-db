1. **Anchor-binding pipeline** (`identity/binding.ts`, `identity/anchorRegistry.ts`) — pluggable, fail-closed `AnchorBinder`s (DOMAIN DNS-01, EMAIL round-trip) emit signed, expiring `AnchorAttestation`s with salted-hash `anchorId`s, deterministic per-anchor `classId`s, and **operator `classId`s that FLEET-CAP** independence (N sources behind one registrar/ASN collapse toward one class). 4. **Undo-engine hardening** (`ratification/disown.ts` + the corroboration / weak-influence / adjudication-provenance / reconcile ledgers) — `downstreamDisownSweep`'s BFS taint-closure over DERIVATION edges (demote-never-delete), the corroboration-event ledger's **exact `reverseCredit`** of recorded deltas, the weak-influence **review queue** (consulted-but-not-cited ⇒ human, never auto-demote), **adjudication re-opening** when a tainted strand's removal collapses a decisive margin, false-disown-as-suppression protection (independently-corroborated derivatives survive), and the total-ledger **reconciliation audit** (off-ledger drift detector). **WIRED into the engine** as the `db.disown(sourceId, opts?)` verb (`api.ts`), which assembles the hardening deps from the wired `RatificationDeps` and runs the whole sweep atomically over the shared handle. (The Merkle witness layer is a standalone tamper-evidence layer composed directly over the same audit chain — kept explicit rather than over-coupling the engine; see `systemCoherence.test.ts` for the composition root.)

Each `LedgerRecord = {seq, prevHash, kind:'PENDING'|'APPROVAL', payload, signerSourceId, thisHash, sig}`; `prevHash` chains (genesis = `sha256("GENESIS")`), `thisHash = sha256(canonical(all fields except thisHash/sig))` over an explicit hand-ordered preimage (never free-form `JSON.stringify`), `sig` = Ed25519 over `utf8(thisHash)` carried base64url. Optional **content-blindness**: PENDING records may carry a `contentHash` fingerprint instead of bodies (the human reviews claims out-of-band). The ledger emits a `ResolvedDispute` **plan**; the engine (`api.ts`) applies the store writes — purity boundary kept (ledger is stateful infra; consolidation stays pure). - **Decisive-or-defer multi-class resolution — IMPLEMENTED** (`forgetting/consolidation.ts`, `tryConsolidate`). What was once a conservative "multi-class always defers" is now a calibrated decisive-or-defer gate: a multi-class (genuinely independent) dispute AUTO-RESOLVES only when reputation — an EXTERNAL signal, never majority/headcount — gives a winner that is both DECISIVE (top−second LCB gap ≥ `decisiveMargin = 0.30`) and EARNED (top ≥ `minWinnerReputation = 0.20`, above the bare-key ceiling); otherwise it DEFERS to the human horn (a weightless flood fails the earned gate; two comparable independents fail the gap). For IRREVERSIBLE decisions the engine passes a `HighImpactContext` (via `db.adjudicate(attr, { highImpact })`) so the decisive margin becomes NECESSARY-BUT-NOT-SUFFICIENT — the winner must ALSO clear ≥2 independent corroborations, a 90-day recency-clean window, and ≥2 disjoint anchor classes, else DEFER no matter the gap.
- **Anchor self-stack ladder cap — IMPLEMENTED + WIRED** (`identity/anchors.ts`, `applySelfStackCap`, now load-bearing). The cap clamps a single source's noisy-OR combined disjoint strength to its OWN strongest single realized `independenceWeight` (`min(combined, max over bindings of independenceWeight)`; 0 for an empty side). Justified against the ladder: a source holding no anchor in class C-or-above has a strongest weight below C's, so a stack of cheaper classes can never reach `ANCHOR_TABLE[C].independenceWeight`. WIRED into `independenceBetween`: each side's `combineSublinear` result is passed through `applySelfStackCap` (over that side's disjoint bindings) before the pair `Math.min`. Real effect: a 10× EMAIL stack (noisy-OR ≈ 0.651) caps to 0.10 and never reaches DOMAIN's 0.35 or HARDWARE's 0.45; a genuine single DOMAIN (0.35) is unaffected; a large FINANCIAL_STAKE bond is honored (the binding's realized weight is the ceiling). Only multi-cheap stacks are clamped.

- **Disown sweep — DOWNSTREAM transitive closure IMPLEMENTED** (`ratification/disown.ts`, `downstreamDisownSweep`). The store-aware orchestrator that finishes pillar 4 ("claws back reputation across everything that anchor ever asserted ... including credit conferred on OTHER sources that used these strands as a witness"). Keeps `identity/reputation.ts`'s ledger primitives pure; the graph traversal lives here where it sees the StrandStore. Flow: (1) delegate the DIRECT-seed crater to `ledger.disownSweep` (authoritative idempotency — an already-disowned source returns an empty seed clawback and we early-return a complete no-op); (2) the TAINTED INDEPENDENCE-CLASS SET = the classes of seed roots whose `sourceId === disowned` (fail-closed fallback to all seed classes if provenance recorded no sourceId); (3) BFS over `store.inEdges(t)` filtered to `EdgeType.DERIVATION` walked BACKWARD (a DERIVATION edge points derived→witness, so the edges ENTERING a tainted witness `t` have `from` = a DERIVED strand that rested on `t` — the downstream frontier); (4) per downstream strand, TWO SEPARATE decisions — DEMOTE (existence-rests-on: mint a synthetic OUTRANKS edge from a deterministic disown sentinel, `demote`, `putStrand` — never delete, archive stub intact) and CONTRADICT (credit-funded-by: `ledger.contradict` the strand's backing sources ONLY if a source has a provenance root in the tainted class set). A DERIVATION-reachable strand backed by a genuinely INDEPENDENT class is demoted but its source is NOT contradicted — coincidental independent agreement is never punished (the bounded requirement). Idempotent (delegated `disowned` set), cycle-safe (`visited` set → each strand once, mutual DERIVATION edges terminate), deterministic (frontier sorted by `(content_hash, id)`, in-edges by edge id), fails-closed (a dangling edge / missing strand skips that node, never aborts), dedupe-by-root (seed + frontier deduped by `content_hash` so a same-root flood counts once → one demotion, one contradiction). Tested per invariant (`ratification/disown.test.ts`, `ratification/disownHardening.test.ts`): derived-from-disowned demoted (+ transitive derived-of-derived); coincidental different-class NOT touched; different-class DERIVATION-reachable demoted-but-not-contradicted; tainted-class source IS contradicted; idempotent second sweep no-op; cycle-safe no-hang; demote-never-delete (content_hash + provenance intact); dedupe-by-root counts once. **CORROBORATION-CREDIT REVERSAL — CLOSED (substrate-backed + engine-wired):** the residual that was once `TODO(crack-A)` (source B earned credit for AGREEING with disowned A but carries no DERIVATION edge to A) is now closed by the **corroboration-event ledger** (`ratification/corroboration.ts`): a corroboration-driven gain is recorded AT EARNING TIME with the EXACT applied α-mass, and the sweep reverses precisely that delta on each beneficiary whose event intersects the tainted seed — bounded, exact, idempotent (`CORROBORATION_CREDIT_SUBSTRATE_SPEC` documents it as CLOSED), never punishing coincidental independent agreement. The whole sweep — direct crater + downstream demotions/edges + class-bounded contradicts + exact credit reversals + weak-influence review queue + adjudication re-opening — runs as ONE atomic transaction over the shared handle and is reachable through the engine's `db.disown(...)` verb.

- **Atomic compound writes — IMPLEMENTED (hardening tick 3, durability final)** (`store/sqliteStore.ts`, `api.ts`, `ratification/disown.ts`). Tick 1 made FACTS durable; tick 2 made TRUST + AUDIT durable; the residual weakness was that a COMPOUND operation is MANY autocommitted writes, so a crash MID-OP could leave a HALF-STATE (a loser demoted with no OUTRANKS edge; an APPROVAL audit record with no matching demotion; a half-finished disown sweep). Closed by making each multi-write engine/ledger operation run inside ONE SQLite transaction over the SHARED db handle. (1) `createSqliteStore` gained a `{ db }` overload so the store can ride the SAME `DatabaseSync` handle as the reputation + ratification ledgers (facts + trust + audit in one crash-consistent file); its `beginTxn()` is NESTABLE via a `#txnDepth` guard (only the outermost issues `BEGIN`/`COMMIT`/`ROLLBACK`; inner `recomputeOutWeightSum` and re-entrant `beginTxn` enroll in the open txn rather than emitting a forbidden nested `BEGIN`), and `close()` is a no-op for a borrowed shared handle (only the owner closes). (2) A `withTxn(store, fn)` helper wraps a compound op: open `beginTxn`, run, `commit`; on throw `rollback` and rethrow — and a NO-OP straight call when the backend has no `beginTxn` (the in-memory store, already atomic-per-call, exactly as the `StoreTxn` contract permits). (3) The four compound ops are wrapped: `adjudicate` RESOLVED (every loser's OUTRANKS edge + demotion + `reputation.contradict` as one unit), `approve` (the signed APPROVAL append + reputation moves + the store persistence of demotions/edges as one unit, so the immortal audit chain never desyncs from the state it describes), `downstreamDisownSweep` (the direct crater + every demotion/edge + every contradict + every precise `reverseCredit`/`markReversed` as one unit), and `writeFact` (the multi-edge sibling attach + both `putStrand`s + every `recomputeOutWeightSum` as one unit). Because the ledgers write through the same handle, their `INSERT`/`UPDATE`s enroll in the open transaction with ZERO signature changes. CORRUPTION DETECTION: `SqliteStrandStore.integrityCheck()` runs `PRAGMA integrity_check` (structural half); the audit chain's `verifyChain` is the semantic half — a flipped byte in a persisted record is named at its first broken seq. Tested (`__tests__/atomicCompound.test.ts`, 6 cases over a shared handle): a forced mid-op throw in adjudicate's 2nd-loser demotion AND in a disown sweep's `contradict` rolls back FULLY (no demotion, no OUTRANKS edge, no rep crater, no audit record; `integrity_check` still ok; the op is cleanly re-runnable and then genuinely RESOLVES 2 demotions); a committed `approve` + a multi-edge `writeFact` survive an UNCLEAN reopen (WAL recovery) with demotions + edges + reputation + signed APPROVAL all present and `verifyChain` ok; `integrity_check` true on a clean db; a tampered persisted audit row → `verifyChain().ok === false` naming seq 1.

### HONEST GAP LIST (post-tick-3) — what IS and is NOT hardened

CRITICAL (would block calling this a "durable single-process prototype") — **none remain**. Atomic compound writes, crash-consistent reopen (WAL recovery leaves no half-applied op), and corruption detection (structural `integrity_check` + semantic chain verifier) were the closing items and are done + tested.

ACCEPTABLE (known, deliberate prototype limitations — each safe for a single-process durable prototype; these are exactly the ARCHITECTURE.md "out-of-scope / operational-social" residuals):
- **Cross-process / concurrent writers:** single-process is assumed. WAL gives ONE writer + many readers; two processes writing the same file concurrently is out of scope (no app-level lock / leader election). A second writer is rejected by SQLite's lock, not coordinated.
- **No REAL external anchor services or witness sinks:** the binder PROVER seams (DNS-01, email round-trip) and the publication sinks are injected ports with in-memory / SQLite test impls; wiring real DNS / WHOIS / email confirmation services and ≥2 genuinely-independent third-party append-only witness logs is the operational step. The Merkle guarantee is DETECTION-given-an-honest-published-anchor, null without live witnesses.
- **HARDWARE / KYC / FINANCIAL_STAKE binders not built:** only DOMAIN and EMAIL binders ship. The anchor-cost table reserves HARDWARE (TPM/WebAuthn), VERIFIED_HUMAN (KYC VC), ORG, and FINANCIAL_STAKE rows with weights/rep_caps, but their concrete binders are not implemented — a source can only reach those classes once they exist.
- **Offline class-assignment liability:** independence CLASS ids are offline-assigned (a human/operator decides which roots are correlated). A class-assignment error silently weakens every downstream bound; this is a standing human-judgment liability the code cannot solve, exactly as ARCHITECTURE.md states.
- **`synchronous=NORMAL` vs `FULL`:** a power-cut/OS crash can lose the LAST committed transaction but can NEVER corrupt the file or leave a half-applied compound op (the txn either fully reached the WAL or did not). `FULL` (fsync on every commit) is the knob for zero-loss-on-power-cut at a throughput cost; `NORMAL` is the deliberate operating point.
- **No backup / restore / point-in-time recovery:** no snapshotting, no WAL-archiving PITR. Recovery is "reopen the file."
- **No encryption-at-rest:** the db file is plaintext JSON rows. (SQLCipher / OS-level FDE is the answer; out of scope here, ZERO-dep constraint.)
- **No schema migration / versioning:** tables are `CREATE TABLE IF NOT EXISTS`; there is no `user_version` ladder. A field change needs a manual migration.
- **Activation-walk-on-crash recovery still STUBBED — but nothing to recover:** the crack-A activation walk is in-memory per-traversal (a transient register, refractory lock, frontier), writing NOTHING mid-walk; a crash mid-recall loses only the in-flight walk and the recovery is "re-issue the cue." There is no persisted half-walk to leave inconsistent, so this is not a durability gap.
- **In-memory / `:memory:` backends are non-durable BY DESIGN:** their `withTxn` is a true no-op (no `beginTxn`); they are the fast default and the test substrate, not the durable store.

Known simplifications to revisit (orthogonal to durability): activation uses path *dominance*, not reinforcement-by-summation; `noveltyOf` is a 0/1 signal (a saturating curve is a tuning knob); `independenceFromStamp` reads `anchor_cost` directly rather than pairwise anchor-set disjointness against the thread's other endpoint.

## What this project is

**Intelligent DB** is a memory substrate for AI agents, built to counter the two ways agents fail: they **forget** (stateless models lose context within and across sessions) and they **hallucinate** (they invent facts and can't tell their own recall from invention).

The design is **not** a vector database. It is a deliberate inversion of one. A vector DB does fuzzy nearest-neighbor lookup with no stable structure, which is exactly why it can't resist hallucination. This design instead models memory as a **memory palace / spider-web**: facts are latent by default and surface only via **spreading activation** along structural threads.

## Core architecture (settled — do not relitigate)

- **Latent memory, activated by traversal, not query.** Nothing sits in a readable list. A cue energizes a seed strand; activation propagates across connected strands until a cluster of relevant facts "lights up." Only lit strands get assembled into an answer.
- **Web of webs.** Dense local webs (one conversation / tight cluster) are loosely bridged to other webs by rare **cross-web bridge** threads. Activation jumping a bridge is what produces "something from last week is suddenly relevant."
- **Threads connect only on: shared entity (mechanical, checkable) + confirmed link (the relationship).** The AI model is the **librarian** that decides where each new strand attaches.
- **Fact states:** `LIVE` (current) / `PROVISIONAL` (a visible superposition held until ratified, e.g. "Berlin→Tokyo pending") / `DEMOTED` (was true, kept as history). Contradiction **demotes, never deletes**. Derived facts like "X moved" *emerge* from one strand outranking another over time.
- **Provenance is first-class** on every strand. Confirmation = corroboration by **independent** provenance. Two strands agreeing from the *same* root is an echo, not corroboration.
- **Observed vs Derived facts.** Observed = from outside (user/doc/tool). Derived = the web computed it by walking itself. **"Wall with a window":** a derived fact may be believed and spoken (with derivation shown) but is **never its own witness** — it only graduates to observed when an external source ratifies it.

### The two governing invariants

1. **The model is never its own witness.** It files and speaks memories; it never confirms them. No provenance → no voice.
2. **The web is never its own witness about source identity** (see Open Problem). Trust in *what* a fact is comes from inside the web; trust in *who* sourced it must come from outside.

## Resolved: traversal halting ("when to stop walking and start speaking")

This crack is **solved** and survived adversarial review. Key mechanics any implementation must preserve:

- **Share-normalized best-first walk.** Pop the max-priority unexpanded strand; `child = parent * (w_edge / Σw_out) * γ`, with `γ≈0.6`, `w_edge = link_confidence * provenance_independence * recency`. Share-normalization makes high-degree junk hubs self-starve (each of N edges gets 1/N). A per-traversal refractory lock kills A→B→A echo. Activation is monotone non-increasing, so termination is provable.
- **Separate ordering from stopping.** Convergence (count of independent-provenance ancestors) may *order* which strand to pop next, but must **never** be the stop gate — a genuine insight bridge has convergence=1 and would be starved.
- **Two-phase stop:** (1) local saturation when new independent corroboration falls below ε; (2) **mandatory bridge sweep** — every lit, un-crossed cross-web bridge gets exactly one guaranteed exploratory crossing, funded by a **separate ~20% bridge sub-budget** (so bridge-chasing can't starve the answer), with a circuit-breaker after 2 zero-yield crossings.
- **Hard backstop:** absolute pop-cap (~2000) + wall-clock; on trip, **stamp `truncated` / `bridge-starved`** — never a silent stop. Halting **fails open** (surfaces low-corroboration strands with a stamp rather than hiding them).

## Resolved floor: forgetting

- Forgetting = **downward tier movement** `HOT→WARM→COLD→ARCHIVE-STUB`, **never deletion** while any provenance edge points at a strand. The archive stub (content hash + independent roots + timestamps) is immortal.
- **Decay sets pressure; gates set permission.** Eviction below COLD requires ALL of: low echo-discounted unique value vs *independent* neighbors; fresh independence stamp; not the outranked side of an active contradiction; not an earned bridge; temporally-discounted independent-source count ≤ 1; past a grace floor.
- New observed strands are pinned WARM for an un-forgeable grace window. Same-root floods collapse to multiplicity 1. A majority vote may only collapse *agreeing same-root echoes* — it may **never** pick a winner among disagreeing independent roots.

## The hard theorem (why the external layer is mandatory)

A multi-agent adversarial council proved that **claim adjudication cannot be solved from inside the graph**:

> Under "identity is priced, not prevented" (a patient attacker can pay a finite cost to mint independent-looking sources), there is **no purely internal rule** that both (a) lets one true witness overturn a planted false canonical AND (b) stops two fake sources overturning a true incumbent. The two requirements are in direct tension.

Three fatal attacks with no *internal* repair:
1. **Contradiction-set bomb** — drip in 500 individually-plausible observed claims about one attribute; each is individually un-prunable and none reaches quorum, so the answer drowns in honest-looking noise the system is forbidden to forget. Denial-of-answer at linear cost.
2. **First-arrival trap** — any incumbent-protecting threshold protects whoever arrived first, and the attacker controls arrival order.
3. **No second independent lock** — every candidate (path-disjointness, age axis) was either correlated with the first lock or anti-correlated with the threat.

The web can judge **what** a fact is; it cannot judge **who** is really behind it. Independence is a property of identity, and identity is forgeable from inside the graph. Therefore identity must be witnessed from **outside** — the Source-Identity Layer below.

## Source-Identity Layer (designed)

Mental model: **passport control at the border of the memory.** Before any source counts as a distinct witness, it shows ID at the door. The web stops *guessing* whether two sources are independent and *asks this layer*, which roots the answer in real-world-scarce things that are expensive to fake. Four pillars:

1. **Passport — cryptographic key per source.** Proves *sameness*: same key = same source, unforgeably. Lets the web collapse echoes (two facts from one key are never corroboration). Cheap to mint, so necessary but **not sufficient** for independence.
2. **Anchor — independence measured against scarce external roots, never declared.** Each key binds to one or more **anchors** (domain, verified human, org, device attestation, posted stake) — things costly/rate-limited in the real world. Independence between two sources = **disjointness of their anchor sets**, weighted by anchor cost. Same anchor → echo; different *costly* anchors → genuinely independent. This is "identity is priced, not prevented" made mechanical: the web is handed the *price* and weighs cheap identities as cheap.
3. **Credit score — reputation, earned slowly, lost fast.** Weight comes from track record (past claims later ratified by independent anchors vs contradicted), not headcount. Kills the **contradiction-bomb** (500 fresh sources start near-zero, loud but weightless) and the **first-arrival trap** (canonical follows earned trust, not arrival order).
4. **Security deposit — staking makes lying cost something.** Asserting (especially *contradicting*) requires a posted stake. A bogus claim burns the stake and, via the existing "disown a bad source" retroactive sweep, claws back reputation across everything that anchor ever asserted. Attacking gets *more* expensive the more you do it.

### Interface to the web

For every incoming observed fact the layer emits a stamp: `{ source_id, anchor_set, anchor_cost, reputation, stake_posted }`. The two quantities the web **cannot compute about itself** — `provenance_independence` (the per-edge halting weight) and the "independent root count" used by the forgetting layer — **stop being self-computed and are read from this stamp instead.** Everything else in the web (activation, traversal, demotion, consolidation) is unchanged.

This honors both invariants: identity comes from outside (web is not its own identity witness), and the layer is **mechanical** — keys, anchors, scores, stakes — so the model still never witnesses.

### Anchor-cost table (product decision — tune to threat model)

`independence_weight` is what one anchor of this class contributes to the independence score between two sources whose anchor sets are disjoint. `rep_cap` is the maximum reputation a source rooted *only* in this class may ever earn (a ceiling, so a cheap anchor can never climb to high trust no matter how long it behaves). Costs are illustrative starting points, not fixed.

| Anchor class | What it proves | Cost to mint another | independence_weight | rep_cap | Notes |
|---|---|---|---|---|---|
| **None (bare key)** | sameness only | ~free | 0.00 | 0.05 | Echo-collapse only; carries no independence. Default for anonymous input. |
| **Email / OAuth account** | a controlled inbox | ~free, rate-limited | 0.10 | 0.30 | Disposable domains weighted to ~0; major-provider OAuth higher. |
| **Phone / SIM** | a provisioned number | low ($), rate-limited | 0.20 | 0.40 | VOIP ranges discounted; carrier-verified higher. |
| **Domain (DNS + TLS)** | control of a registered name | $/yr, registrar-throttled | 0.35 | 0.60 | Age + independent inbound links raise effective weight. |
| **Hardware attestation** | a distinct real device | device $ + enclave | 0.45 | 0.65 | TPM/secure-enclave signature; resists pure-software Sybil. |
| **Verified human (KYC)** | a distinct legal person | high, identity-checked | 0.70 | 0.90 | One human ≈ one anchor; collusion still possible but not free. |
| **Organization / institution** | an accountable entity | high, slow | 0.75 | 0.92 | Reputation-bearing; misbehavior has out-of-band consequences. |
| **Financial stake / bond** | skin in the game | scales with $ locked | 0.30–0.85 (∝ stake) | 0.85 | Weight scales with deposit size; burns on falsity. Composable with any row above. |
| **External authority / notarized event** | a real-world fact signed by a trusted third party | highest | 0.90 | 0.98 | The window in "wall-with-a-window": ratifies derived → observed. |

**Rules for the table:**
- **Independence is set-disjointness, not addition.** Two sources sharing *any* anchor are not independent on that anchor; their independence is driven by their *non-overlapping* costly anchors. Don't let a source self-stack ten cheap anchors to fake one expensive one — independence between two sources caps at the cost of their cheapest *shared* root being absent, and a single source's total is sublinear in its own anchor count.
- **rep_cap binds the ceiling; reputation still has to be earned up to it.** A KYC source is not born trusted — it can merely *eventually* reach 0.90.
- **Stake composes multiplicatively with the row it backs**, so "KYC + large bond" is the strongest practical witness short of an external authority.
- The whole table is **one swappable trust root** (like a Certificate Authority set). Changing which anchor classes you accept and their weights *is* the security policy. This is the intended knob — not a wall to break.

### Residual assumption (state it in any threat model)

This does **not** make Sybil impossible — it converts an unbounded, free attack into a priced, visible, self-limiting one. The anchors themselves are an external trust root (DNS, KYC, attestation authority, staking currency), exactly where TLS and CAs bottom out. The achievement is **shrinking all trust assumptions down to this one small, well-understood, swappable root** instead of scattering them through the graph.

## Strand data model (council-converged fields)

Any implementation of a strand should carry at least:
- activation register (per-traversal) + `refractory_until` timestamp
- incremental ancestor-sketch (MinHash/HLL over offline-assigned independence-class ids) → `convergence_factor`, **for ordering only**
- edge weights `w = link_confidence * provenance_independence * recency` + `out_weight_sum`
- `edge_type` with explicit `CROSS-WEB-BRIDGE` marker (typed structural class)
- `earned_bridge_value` + `far_side_potential` (accrue only from crossings ratified by ≥2 independent roots; recomputed offline — the query stream cannot write them)
- bridge sub-budget counter + low-yield-crossing counter
- provenance root-set with per-root independence-class + establishment timestamp
- salience `s` with `last_fire_time`, decay `λ`, real-retrieval `fire_count`
- `description_value` (reconstruction-loss bits vs independent neighbors; echo-discounted) — consolidation-eligibility gate only
- `fact_state` enum + observed/derived flag + the single outranking edge that explains a demotion
- observation-age (grace floor) + external-re-observation-count
- contradiction-set membership id + per-(entity,attribute) co-equal-claim cardinality
- `content_hash` + cold-store backpointer; `tier` enum; `reason_code` stamped on every stop/tier-move

## Design philosophy for future work

- Prefer **structural** defenses (a property an attack mathematically can't satisfy) over **policy** defenses (a threshold an attacker can tune around). Share-normalization beating hub spam is the template.
- When tempted to let the model decide something load-bearing (when to stop, what's canonical, what's independent), **don't** — route it to a gate, a provenance check, or an external signal. The model proposes; it never witnesses.
- Treat every new keep/prune/promote rule as adversary-facing: ask "what does a patient attacker who can pay to mint identities do to this?" before trusting it.
