1. **Crypto-free trust registry** (`identity/trustRegistry.ts`, `identity/sources.ts`) — identity is CONSUMED from configuration, never manufactured or proven by this codebase. Plain `SourceRef`s carry a deterministic `sourceIdFor(issuer, subject)` checksum id (sameness only), and four claim producers mint priced anchors: `registerOwner` (the PERSONAL preset's ground truth, OWNER at external-authority grade), `registerSsoMember` (SSO_TENANT_MEMBER — deliberately email-grade since a fresh tenant is near-free; a DOMAIN lift ONLY when the registry CONFIG verifies the tenant's custom domain — a caller hint that is unconfigured or contradicts config grants nothing), `registerPublisher` (URLs collapse to eTLD+1 ⇒ one source; PUBLISHER_TRACKED only for configured publishers; an `operatorOf` fleet axis so N domains behind one operator collapse), and `registerSystemOfRecord` (registry-configured authoritative systems, e.g. Workday-for-HR). **Operator `classId`s FLEET-CAP** independence exactly as before, and `independentSources` fails CLOSED for unregistered/anchorless sides.
2. **Tamper-evident checksum chain** (`ratification/pendingLedger.ts`) — the append-only audit ledger: each record's `thisHash` is a plain SHA-256 checksum over an explicit hand-ordered preimage, `prevHash` chains from a fixed genesis, `verifyChain()` recomputes the whole chain and names the first broken seq (any at-rest byte-flip is caught), and `chainHead()` exports an O(1) `{seq, headHash}` checkpoint for ACCESS-SEGREGATED external storage. DISCLOSED TRADE-OFF (the deliberate Phase-2 trade-off, executable in `cryptoFreeTrust.test.ts`): `signerSourceId` is ASSERTED attribution committed into the checksum — an actor with live WRITE access can rewrite history from any seq, recompute every checksum, and read green; what they canNOT do is reproduce a previously exported checkpoint (even a same-length rewrite diverges at `headHash`). Detection against an insider therefore rests on getting copies somewhere the writer can't touch, via two composable plain-data mechanisms: scheduled `chainHead()` checkpoints, and **real-time shipping** — an optional `AppendSink` passed at ledger construction (or `createAgentMemory({ onLedgerAppend })`) receives every record BEFORE the local write (ship-before-write: a throwing sink aborts the append fail-closed; a shipped-then-rolled-back record is the safe, auditor-resolvable direction), so a local rewrite diverges from the already-shipped copy at the first rewritten seq — executable end-to-end in `auditShipping.test.ts` (a `verifyChain`-clean insider rewrite is caught at exactly the forged seq). Reference sink implementations (append-only JSONL mirror, the crash-safe SPOOL pattern for network destinations like a SIEM, the explicit `lenient` availability wrapper, and the `firstDivergence` detection comparator) ship typechecked + tested in `src/examples/auditSinks.ts` — deliberately NOT exported from the barrel (the API is the `AppendSink` type; destinations belong to the deployment). See KNOWN LIMITATIONS.
3. **Beta(α,β) trust scoring** (`identity/reputation.ts`) — independence-weighted α (headcount denied at the caller, one `ratify` per class), 4× asymmetric β, **90-day exponential decay applied ON EACH READ** (pure decay-on-read: a dormant high-LCB source reads stale-discounted IMMEDIATELY, not only at the next mutation — side-effect-free, never persisted; decay-before-mutate still governs writes), lower-confidence-bound readout (prior reads EXACTLY 0), `rep_cap` ceiling, exact `reverseCredit` — including unwinding the non-decaying `corroborationDepth` floor it pins (RE-EXACTED 2026-07-07: the ORIGINAL 2026-07-07 fix subtracted the same recorded α-mass `w` from `corroborationDepth` too, which only genuinely unwinds the floor at the narrow boundary where that happens to cross `floorDeadband` — a hostile re-audit independently recomputed from source constants that a realistic well-corroborated disown, e.g. depth=10, only moved the LCB from 0.7839 to 0.7654, and that the same scalar subtraction could erode an UNRELATED, still-valid corroboration's own floor. Root cause: `α` is an ADDITIVE evidence mass while `corroborationDepth` is a `Math.max` over MIS-depth SNAPSHOTS — not commensurable, so subtracting one from the other by the same amount is not "exact" in general. Fixed by recording the raw MIS depth each corroboration event snapshotted (`CorroborationEvent.corroborationDepthAtEvent`) and, on reversal, recomputing the beneficiary's TRUE surviving floor as the MAX snapshot over every OTHER, still-unreversed event for that beneficiary (`CorroborationLedger.eventsByBeneficiary`, `ReputationLedger.reverseCredit`'s new `newCorroborationDepth` parameter) — see `ratification/depthFloorReversal.test.ts`'s parametrized depth={2,4,6,8} case and its collateral-erosion case, both driving the real `ratify`/`disown`/`scoreOf` verbs) — and a decisive-or-defer adjudicator with a high-impact gate (count + recency + ≥2 anchor classes on top of the LCB margin).

- **Contradiction adjudication — IMPLEMENTED, theorem-faithful** (`forgetting/consolidation.ts`, `tryConsolidate`). Signature now `tryConsolidate(set, members, stampsByRoot, now, mintEdgeId?)` returning `ConsolidationOutcome = {kind:'RESOLVED', demotions} | {kind:'DEFERRED', pending} | {kind:'NOOP'}`. **Single independence class** (echo dispute, the safe case) → resolved by *external stamp signal only*, lexicographically `reputation → anchor_cost → stake_posted → deterministic id` (`stake_posted` is a frozen stamp-shape field, constant 0 since staking's retirement), **never majority/headcount**; losers demoted (DEMOTED + `outranked_by`, never deleted). **Multiple independence classes** (genuine independent dispute) → the safety gate fires *first*: it auto-resolves ONLY on a decisive, earned reputation margin (see "Decisive-or-defer" below), else it `DEFERRED`s, emitting a `PendingRatification {contradictionSetId, attribute, members (reputation-ranked), reason:'INDEPENDENT_DISPUTE', createdAt}` — the external **human ratify horn** (Tanaka-shaped second-admin); the web never picks an in-graph winner by majority/headcount. - **Backing the `PendingRatification` horn — IMPLEMENTED** (`ratification/pendingLedger.ts`, wired in `api.ts`). Tanaka-shaped, native TS ("a vault and a doorbell, never a judge"). THE VAULT: an append-only, hash-chained ratification ledger — a tamper-evident CHECKSUM chain, no keys — behind a swappable `PendingLedger` interface (in-memory + SQLite impls). Each `LedgerRecord = {seq, prevHash, kind:'PENDING'|'APPROVAL'|'MUTATION', payload, signerSourceId, thisHash}`; `prevHash` chains (genesis = `sha256("GENESIS")`), `thisHash = sha256(canonical(all fields except thisHash))` over an explicit hand-ordered preimage (never free-form `JSON.stringify`); `signerSourceId` is ASSERTED attribution committed into the checksum (see the attribution trade-off in KNOWN LIMITATIONS). A standalone `verifyChain(): {ok, firstBrokenSeq}` walks the chain, recomputes every hash, and names the first broken seq (the "money artifact" — flip any persisted byte and it reports `ok:false`); `chainHead(): {seq, headHash}` is the O(1) checkpoint to ship to access-segregated storage. Optional **content-blindness**: PENDING records may carry a `contentHash` fingerprint instead of bodies (the human reviews claims out-of-band). THE DOORBELL: `appendPending(pending, systemSource)` records a PENDING; `listPending()` returns open disputes (reputation-ranked) for a reviewer; `approve(csid, winner, approver: SourceId, now, ctx)` enforces the **distinct-approver gate** (rejects self-approval — the approver must NOT have authored any disputed member), a **provenance gate** (the approver must be registered with ≥1 priced anchor, fail-closed — "no provenance → no voice"), and the **RC-5 anchor-disjointness gate** (the approver must be `independentSources`-independent of every disputed author), then records an APPROVAL and RESOLVES: mint OUTRANKS winner→each loser, `demote()` the losers (DEMOTED + outranked_by, never deleted), drive reputation (winner ratified, losers contradicted). The ledger emits a `ResolvedDispute` **plan**; the engine (`api.ts`) applies the store writes — purity boundary kept (ledger is stateful infra; consolidation stays pure). Engine wiring: `adjudicate(attribute)` resolves LIVE members, builds `stampsByRoot` from the identity layer, runs `tryConsolidate`, and routes RESOLVED→persist demotions+OUTRANKS+`reputation.contradict`, DEFERRED→`appendPending` (throws if no ledger is wired — a deferral is never silently dropped), NOOP→nothing. `listPending()`/`approve()` are exposed on `IntelligentDb`; `createIntelligentDb(..., ratification?: RatificationDeps)` is optional/back-compatible. - **Decisive-or-defer multi-class resolution — IMPLEMENTED** (`forgetting/consolidation.ts`, `tryConsolidate`). What was once a conservative "multi-class always defers" is now a calibrated decisive-or-defer gate: a multi-class (genuinely independent) dispute AUTO-RESOLVES only when reputation — an EXTERNAL signal, never majority/headcount — gives a winner that is both DECISIVE (top−second LCB gap ≥ `decisiveMargin = 0.30`) and EARNED (top ≥ `minWinnerReputation = 0.20`, above the bare-key ceiling); otherwise it DEFERS to the human horn (a weightless flood fails the earned gate; two comparable independents fail the gap). For IRREVERSIBLE decisions the engine passes a `HighImpactContext` (via `db.adjudicate(attr, { highImpact })`) so the decisive margin becomes NECESSARY-BUT-NOT-SUFFICIENT — the winner must ALSO clear ≥2 independent corroborations, a 90-day recency-clean window, and ≥2 disjoint anchor classes, else DEFER no matter the gap.
- **Dispute horn surfaced per tier — IMPLEMENTED (Phase 4)** (`agent/agentMemory.ts`, `mcp/handler.ts`, `ratification/disputeRouting.ts`, `ratification/pendingLedger.ts`). PERSONAL: the agent facade now WIRES a pending ledger (a deferral is recorded, never thrown) and renders every open dispute as a plain-data question — `pendingQuestions()` (members resolved fail-closed from the store; a dangling id renders nothing) answered by `resolvePending(csid, chosenStrandId)`, exposed over MCP as the `list_pending_questions` / `resolve_pending` tools. **Owner-override policy hook:** `resolvePending` calls `approve()` under `ApproveContext.allowAuthorApprover` (threaded via `engine.approve(..., opts?: ApproveOptions)`) — default absent/false, enterprise semantics byte-identical — which bypasses ONLY (1) the distinct-approver (self-approval) gate and (2) the RC-5 anchor-independence-vs-authors check; dispute-open, winner-is-member, and registered-with-anchors stay UNCONDITIONAL (even the owner shows a priced anchor). WHY it is not a weakening: the owner is the personal tier's trust root (EXTERNAL_AUTHORITY-grade OWNER anchor, rep_cap-0.98 tier, no second admin in a mom-and-pop deployment), and the owner overriding a memory they authored ("you told me X in March") is ground truth, not self-dealing. Every override is RECEIPTED: the APPROVAL record carries `ownerOverride: true`, committed into the checksum (emit-only-when-present, legacy hashes byte-stable) — auditable forever. A dangling loser mid-resolution is SKIPPED fail-closed by the ledger's approve loop (`memberStrand` may return null — mirroring the disown sweep's missing-strand rule), so a degraded dispute stays answerable instead of wedging open on every retry. ENTERPRISE: `createDisputeRouter(config)` — a pure, zero-I/O, deterministic first-match-wins routing adapter mapping open `PendingPayload`s to owning-group labels (IdP groups), with optional `highImpactAssignTo` escalation and a FAIL-CLOSED `entityPrefix` criterion (matches only caller-resolved entity evidence); transport is deliberately the deployment's job. QUARANTINE INTERACTION: disputes only ever form among LIVE strands (`adjudicate` admits only LIVE members), so the horn surfaces genuine believed-fact conflicts — a PROVISIONAL flood produces ZERO pending questions. MCP-BOUNDARY HARDENING (post-Phase-4 review fixes): the recall tool's rendering LABELS non-LIVE states (`[DEMOTED]` / `[PROVISIONAL]` — the `CitedFact.fact_state` invariant carried to the last rendered hop), and both renderers ESCAPE control characters + quote-delimit untrusted payload text so a hostile fact can never forge the option/strandId line structure the relaying agent echoes into the state-mutating `resolve_pending`. Tested (`__tests__/disputeHorn.test.ts`): happy path, override scope (raw self-approval still rejected; anchorless rejected even WITH the flag), quarantine exclusion, MCP round-trip + DEMOTED labeling, routing determinism/purity, dangling-member answerability, injection resistance.

- **Anchor self-stack ladder cap — IMPLEMENTED + WIRED** (`identity/anchors.ts`, `applySelfStackCap`, now load-bearing). The cap clamps a single source's noisy-OR combined disjoint strength to its OWN strongest single realized `independenceWeight` (`min(combined, max over bindings of independenceWeight)`; 0 for an empty side). Justified against the ladder: a source holding no anchor in class C-or-above has a strongest weight below C's, so a stack of cheaper classes can never reach `ANCHOR_TABLE[C].independenceWeight`. WIRED into `independenceBetween`: each side's `combineSublinear` result is passed through `applySelfStackCap` (over that side's disjoint bindings) before the pair `Math.min`. Real effect: a 10× EMAIL stack (noisy-OR ≈ 0.651) caps to 0.10 and never reaches DOMAIN's 0.35 or HARDWARE's 0.45; a genuine single DOMAIN (0.35) is unaffected; the FINANCIAL_STAKE math is honored where a binding exists (an inert data row since staking's retirement — no producer mints it). Only multi-cheap stacks are clamped.

- **Disown sweep — DOWNSTREAM transitive closure IMPLEMENTED** (`ratification/disown.ts`, `downstreamDisownSweep`). The store-aware orchestrator that finishes pillar 4 ("claws back reputation across everything that anchor ever asserted ... including credit conferred on OTHER sources that used these strands as a witness"). Keeps `identity/reputation.ts`'s ledger primitives pure; the graph traversal lives here where it sees the StrandStore. Flow: (1) delegate the DIRECT-seed crater to `ledger.disownSweep` (authoritative idempotency — an already-disowned source returns an empty seed clawback and we early-return a complete no-op); (2) the TAINTED INDEPENDENCE-CLASS SET = the classes of seed roots whose `sourceId === disowned` (fail-closed fallback to all seed classes if provenance recorded no sourceId); (3) BFS over `store.inEdges(t)` filtered to `EdgeType.DERIVATION` walked BACKWARD (a DERIVATION edge points derived→witness, so the edges ENTERING a tainted witness `t` have `from` = a DERIVED strand that rested on `t` — the downstream frontier); (4) per downstream strand, TWO SEPARATE decisions — DEMOTE (existence-rests-on: mint a synthetic OUTRANKS edge from a deterministic disown sentinel, `demote`, `putStrand` — never delete, archive stub intact) and CONTRADICT (credit-funded-by: `ledger.contradict` the strand's backing sources ONLY if a source has a provenance root in the tainted class set). A DERIVATION-reachable strand backed by a genuinely INDEPENDENT class is demoted but its source is NOT contradicted — coincidental independent agreement is never punished (the bounded requirement). Idempotent (delegated `disowned` set), cycle-safe (`visited` set → each strand once, mutual DERIVATION edges terminate), deterministic (frontier sorted by `(content_hash, id)`, in-edges by edge id), fails-closed (a dangling edge / missing strand skips that node, never aborts), dedupe-by-root (seed + frontier deduped by `content_hash` so a same-root flood counts once → one demotion, one contradiction). Tested per invariant (`ratification/disown.test.ts`, `ratification/disownHardening.test.ts`): derived-from-disowned demoted (+ transitive derived-of-derived); coincidental different-class NOT touched; different-class DERIVATION-reachable demoted-but-not-contradicted; tainted-class source IS contradicted; idempotent second sweep no-op; cycle-safe no-hang; demote-never-delete (content_hash + provenance intact); dedupe-by-root counts once. **CORROBORATION-CREDIT REVERSAL — BOUNDED (exact over the recorded closure; substrate-backed + engine-wired):** the residual that was once `TODO(crack-A)` (source B earned credit for AGREEING with disowned A but carries no DERIVATION edge to A) is addressed by the **corroboration-event ledger** (`ratification/corroboration.ts`): a corroboration-driven gain is recorded AT EARNING TIME with the EXACT applied α-mass, and the sweep reverses precisely that delta on each beneficiary whose recorded event intersects the **full demoted taint closure** (seed ∪ demoted-downstream — F2, not just the one-hop seed), never punishing coincidental independent agreement. The reversal is **exact over the recorded DERIVATION + corroboration closure**, NOT "disown is exact" in the unqualified sense: re-observed / uncited influence (re-observation laundering, bond-withdrawal credit, coincidental agreers with no recorded funding link) is a **priced-not-prevented residual, reported SAFE-DEFER, not DEFENDED** (`CORROBORATION_CREDIT_SUBSTRATE_SPEC` documents it as BOUNDED). The intersection IS the guard (F3): reversing on agreement alone would manufacture a NEW false positive, so credit is reversed only where the engine-derived agreement set intersects the demoted closure. The whole sweep — direct crater + downstream demotions/edges + class-bounded contradicts + exact credit reversals + weak-influence review queue + adjudication re-opening — runs as ONE atomic transaction over the shared handle and is reachable through the engine's `db.disown(...)` verb.

- **Atomic compound writes — IMPLEMENTED (hardening tick 3, durability final)** (`store/sqliteStore.ts`, `api.ts`, `ratification/disown.ts`). Tick 1 made FACTS durable; tick 2 made TRUST + AUDIT durable; the residual weakness was that a COMPOUND operation is MANY autocommitted writes, so a crash MID-OP could leave a HALF-STATE (a loser demoted with no OUTRANKS edge; an APPROVAL audit record with no matching demotion; a half-finished disown sweep). Closed by making each multi-write engine/ledger operation run inside ONE SQLite transaction over the SHARED db handle. (1) `createSqliteStore` gained a `{ db }` overload so the store can ride the SAME `DatabaseSync` handle as the reputation + ratification ledgers (facts + trust + audit in one crash-consistent file); its `beginTxn()` is NESTABLE via a `#txnDepth` guard (only the outermost issues `BEGIN`/`COMMIT`/`ROLLBACK`; inner `recomputeOutWeightSum` and re-entrant `beginTxn` enroll in the open txn rather than emitting a forbidden nested `BEGIN`), and `close()` is a no-op for a borrowed shared handle (only the owner closes). (2) A `withTxn(store, fn)` helper wraps a compound op: open `beginTxn`, run, `commit`; on throw `rollback` and rethrow — and a NO-OP straight call when the backend has no `beginTxn` (the in-memory store, already atomic-per-call, exactly as the `StoreTxn` contract permits). (3) The compound ops are wrapped: `adjudicate` RESOLVED (every loser's OUTRANKS edge + demotion + `reputation.contradict` as one unit), `approve` (the APPROVAL append + reputation moves + the store persistence of demotions/edges as one unit, so the immortal audit chain never desyncs from the state it describes), `downstreamDisownSweep` (the direct crater + every demotion/edge + every contradict + every precise `reverseCredit`/`markReversed` as one unit), `writeFact` (the multi-edge sibling attach + both `putStrand`s + every `recomputeOutWeightSum` as one unit), and `ratify` (strand promotion + the reputation credit + the corroboration-event record + the `assertRatifyEmitsEvent` check as one unit — **`ratify` was missed in tick 3 and fixed post-hoc**, see KNOWN LIMITATIONS: it delegates to a private `#ratifyImpl` wrapped in the same `withTxn` helper as the other four, closing a window where a crash mid-ratify — or the corruption-detecting `assertRatifyEmitsEvent` throw itself — could leave a strand promoted with no matching reputation credit). Because the ledgers write through the same handle, their `INSERT`/`UPDATE`s enroll in the open transaction with ZERO signature changes. FIX uncovered: `approve`'s persistence re-read a FRESH (un-demoted) strand on a clone-on-read backend (SQLite), so the demotion was never written — now the engine persists the EXACT object the ledger mutated in place (cached in `handed`), correct under both backends. **A second desync, closed 2026-07-07:** all of the above assumes the store, reputation ledger, and pending ledger share one `DatabaseSync` handle — but `agent/agentMemory.ts`'s `createAgentMemory({dbPath})`, the actual default most callers construct, wired the pending ledger IN-MEMORY regardless of `dbPath`, so this atomicity guarantee never applied to the facade people actually use; a forced mid-`approve()` throw against the real, unmodified factory left the dispute silently half-resolved. Fixed: the default facade now owns one shared handle across all three, plus a `resyncIndex()` recovery path for the pending ledger's open-dispute index after a rollback. Regression test built against the real factory, not a hand-wired rig (`__tests__/approveDesyncDefaultFacade.test.ts`): arms a one-shot failure on a real `INSERT INTO edges`, forces a genuine mid-`approve()` throw, and confirms the dispute stays open, the loser stays undemoted, and a fresh independent reopen of the file verifies clean with zero persisted APPROVAL records. CORRUPTION DETECTION: `SqliteStrandStore.integrityCheck()` runs `PRAGMA integrity_check` (structural half); the audit chain's `verifyChain` is the semantic half — a flipped byte in a persisted record is named at its first broken seq. Tested (`__tests__/atomicCompound.test.ts`, 7 cases over a shared handle): a forced mid-op throw in adjudicate's 2nd-loser demotion AND in a disown sweep's `contradict` rolls back FULLY (no demotion, no OUTRANKS edge, no rep crater, no audit record; `integrity_check` still ok; the op is cleanly re-runnable and then genuinely RESOLVES 2 demotions); a committed `approve` + a multi-edge `writeFact` survive an UNCLEAN reopen (WAL recovery) with demotions + edges + reputation + the APPROVAL record all present and `verifyChain` ok; `integrity_check` true on a clean db; a tampered persisted audit row → `verifyChain().ok === false` naming seq 1; a forced mid-op throw in `ratify()`'s `reputation.ratify` call rolls back FULLY (strand stays DERIVED/PROVISIONAL, provenance unchanged, zero reputation gain, zero corroboration events) and a clean re-run afterward genuinely succeeds.

- **Relay fix — causal-origin independence classes** (`api.ts`, `core/types.ts`, `ratification/disown.ts`). Closes the RELAY-LAUNDERING hole: the write path minted every strand's independence class from the FILING agent alone (`class:${source_id}`), so agent A researching a fact and agent B re-filing what A told it in-context produced TWO classes for ONE observation — manufactured corroboration with zero attacker, trusted by the MIS count, the high-impact gate, and the depth floor. `WriteFactInput` gains an OPTIONAL `causalOrigin: CausalOrigin` (`TOOL_CALL`/`DOCUMENT` with a caller-canonicalized `resourceId`, `USER_STATEMENT`, or `AGENT_RELAY` with `consultedStrandIds`). Class rules, all at WRITE TIME so the existing Bron–Kerbosch `independentRootCount` does the rest UNMODIFIED (no new reachability check anywhere): omitted or `USER_STATEMENT` ⇒ today's behavior EXACTLY (string-equal `class:${source_id}` fallback — every pre-fix caller is bit-for-bit unaffected); `TOOL_CALL`/`DOCUMENT` ⇒ a deterministic per-resource class `class:resource:sha256(kind + NUL + resourceId)` so the SAME resource collapses to ONE class regardless of which agent fetched it (kind participates: domain separation); `AGENT_RELAY` ⇒ COPY the resolved consulted strands' classes (one minted root per DISTINCT upstream class; fresh rootId; sourceId = the FILING stamp; `inheritedClass: true`) plus one deterministic-id DERIVATION edge per witness (derived→witness, the direction `downstreamDisownSweep`'s `inEdges` BFS expects), all inside `writeFact`'s existing `withTxn`; `writeFactsBatch` shares the identical mint path. Fail-safe edges: empty/all-unresolvable consulted lists fall back to the default class (identical to omission, no dangling edges); duplicate ids dedupe; Σw reconciled via one `recomputeOutWeightSum`. Two adversarial findings fixed before landing: (1) the ECHO GATE — a witness's class is copied ONLY for the SAME CLAIM (same `content_hash` + same `attribute`), so a contradicting payload citing a rival cannot inherit the rival's class and collapse a genuine multi-class dispute into the single-class echo-resolve lane (refused witnesses keep their DERIVATION citation but contribute no class); (2) the taint bound — `ProvenanceRoot.inheritedClass` marks relay/resource-copied classes, and the disown sweep's tainted-class set EXCLUDES them (the class belongs to the upstream witness/resource), so disowning a relayer can no longer contradict-and-scar the honest upstream source's whole class (the clawback still fires in full for classes the fraudster OWNED). Tested (`__tests__/relayFix.test.ts`, 23 tests over both backends): headline laundering, transitive relay, same-resource collapse, genuine-independence positive control, fallback exactness, fail-safe edges, SQLite mid-op rollback, batch parity, disown-a-relayer taint bound, echo gate.

- **Trust-tiered ingest — the quarantine gate (Phase 3)** (`api.ts`, design doc §4.1). Closes the fail-open ingest hole: `makeObservedStrand` used to mint EVERY observed strand `LIVE` regardless of who filed it, so an anonymous/near-noise source's claim was believed on arrival. Now `writeFact`/`writeFactsBatch` gate each fact through `#ingestStateFor(filer)`: the ENGINE re-stamps the filer via `identity.stampFor` (engine-owned evidence — the caller-supplied `stamp.anchor_set` is inflatable and is never trusted for the gate, mirroring `ratify`'s discipline) and takes the stamp's STRONGEST single anchor `independenceWeight`. Below `quarantineThreshold` ⇒ the fact lands `PROVISIONAL`/WARM — the existing visible-superposition state, no new enum; at-or-above ⇒ `LIVE`/WARM as before. THE THRESHOLD TABLE (default `DEFAULT_QUARANTINE_THRESHOLD = 0.10`): BARE_KEY (0.00) and PUBLISHER_UNVERIFIED (0.04) quarantine; SSO_TENANT_MEMBER (0.12), PUBLISHER_TRACKED (0.18), LOCAL_DOCUMENT (0.35), OWNER / SYSTEM_OF_RECORD (0.90) do not. The knob is `createIntelligentDb(..., ingest?: IngestPolicy)` — OMITTED means the gate is ON at 0.10 (fail-open-forever was the bug); `quarantineThreshold: 0` is the EXPLICIT escape hatch restoring legacy always-LIVE. A quarantined strand is otherwise byte-identical to LIVE ingest (same WARM grace pin, entity indexing, real `description_value`), stays traversable/recallable — labeled, never hidden (the facade's `CitedFact` carries `fact_state` so the consuming agent can tell a held superposition from a believed fact) — but structurally CANNOT demote a LIVE incumbent (`adjudicate` admits only LIVE members ⇒ a PROVISIONAL flood is a `NOOP`) and exits quarantine ONLY through the EXISTING promotion paths: `ratify()` by an anchor-INDEPENDENT external source, or an `approve()` resolution — no new promotion machinery. THE QUARANTINE-EXIT GATE (`#independentOfProvenance`): the ratifier must be `identity.independentSources`-independent of EVERY source on the strand's provenance (the same RC-5 predicate the approve-gate uses — one independence notion, anti-drift; fail-closed for unregistered/anchorless sides and empty provenance). An ECHO ratify (the author itself, or a fleet/operator-correlated sibling) still appends the external root, still bumps `external_reobservation_count`, still drives reputation — but belief does NOT flip: confirmation is corroboration by INDEPENDENT provenance, and same-root agreement is an echo. A low-trust filer relaying a high-trust strand still quarantines (the relay fix copies the witness CLASS for independence-counting; the SPEAKER's trust gates belief). Tested (`__tests__/quarantineIngest.test.ts`, 13 cases over both backends: tier table, engine-owned evidence vs an inflated caller stamp, PROVISIONAL-flood NOOP, independent-exit vs echo-denied, recall visibility, escape hatch, relay interaction, batch parity, disown interaction; plus the facade `fact_state` regression in `__tests__/agentMemory.test.ts`).

1. **HIGH — `reverseCredit`'s depth-floor unwind only genuinely moved the LCB at the narrow
   depth=2 boundary its sole regression test encoded**, not at a realistic well-corroborated depth.

**Retrieval-quality honesty note (Phase 1 §5-6 measurement gate, 2026-07-06):** an optional embedder (`EmbedderPort`, seed-only — see the Status header above) was measured against a genuine external comparator, mem0, on the real LoCoMo corpus. The gate required the embedder-seeded arm to match mem0's recall@20 (0.484); it measured **0.366, a 0.118 shortfall**, and did not even beat this repo's own plain TunedHybrid bench arm (0.366 vs 0.375) — replacing a simple BFS graph channel with an embedder-seeded activation walk was net-neutral-to-slightly-negative on this dataset, not an improvement. Reported as measured, not tuned to pass. This does not affect the poisoning-defense claims above (Sybil/FactWorld gates re-passed 24/24 / 0.0% ASR with the same embedder configured) — it is a day-to-day ranking-quality gap, not a belief-integrity gap. Full numbers: `docs/BENCHMARK_NARRATIVE.md` §2.1, `BENCH_RERUN_2026-07-06.md`.

## What this project is

**Intelligent DB** is a memory substrate for AI agents, built to counter the two ways agents fail: they **forget** (stateless models lose context within and across sessions) and they **hallucinate** (they invent facts and can't tell their own recall from invention).

The design is **not** a vector database. It is a deliberate inversion of one. A vector DB does fuzzy nearest-neighbor lookup with no stable structure, which is exactly why it can't resist hallucination. This design instead models memory as a **memory palace / spider-web**: facts are latent by default and surface only via **spreading activation** along structural threads.

## Core architecture (settled — do not relitigate)

- **Latent memory, activated by traversal, not query.** Nothing sits in a readable list. A cue energizes a seed strand; activation propagates across connected strands until a cluster of relevant facts "lights up." Only lit strands get assembled into an answer.
- **Web of webs.** Dense local webs (one conversation / tight cluster) are loosely bridged to other webs by rare **cross-web bridge** threads. Activation jumping a bridge is what produces "something from last week is suddenly relevant."
- **Threads connect only on: shared entity (mechanical, checkable) + confirmed link (the relationship).** The AI model is the **librarian** that decides where each new strand attaches.
- **Fact states:** `LIVE` (current) / `PROVISIONAL` (a visible superposition held until ratified, e.g. "Berlin→Tokyo pending") / `DEMOTED` (was true, kept as history). Contradiction **demotes, never deletes**. Derived facts like "X moved" *emerge* from one strand outranking another over time.
- **Provenance is first-class** on every strand. Confirmation = corroboration by **independent** provenance. Two strands agreeing from the *same* root is an echo, not corroboration.
- **Observed vs Derived facts.** Observed = from outside (user/doc/tool). Derived = the web computed it by walking itself. **"Wall with a window":** a derived fact may be believed and spoken (with derivation shown) but is **never its own witness** — it only graduates to observed when an external source ratifies it.

### The two governing invariants

1. **The model is never its own witness.** It files and speaks memories; it never confirms them. No provenance → no voice.
2. **The web is never its own witness about source identity** (see Open Problem). Trust in *what* a fact is comes from inside the web; trust in *who* sourced it must come from outside.

## Resolved: traversal halting ("when to stop walking and start speaking")

This crack is **solved** and survived adversarial review. Key mechanics any implementation must preserve:

- **Share-normalized best-first walk.** Pop the max-priority unexpanded strand; `child = parent * (w_edge / Σw_out) * γ`, with `γ≈0.6`, `w_edge = link_confidence * provenance_independence * recency`. Share-normalization makes high-degree junk hubs self-starve (each of N edges gets 1/N). A per-traversal refractory lock kills A→B→A echo. Activation is monotone non-increasing, so termination is provable.
- **Separate ordering from stopping.** Convergence (count of independent-provenance ancestors) may *order* which strand to pop next, but must **never** be the stop gate — a genuine insight bridge has convergence=1 and would be starved.
- **Two-phase stop:** (1) local saturation when new independent corroboration falls below ε; (2) **mandatory bridge sweep** — every lit, un-crossed cross-web bridge gets exactly one guaranteed exploratory crossing, funded by a **separate ~20% bridge sub-budget** (so bridge-chasing can't starve the answer), with a circuit-breaker after 2 zero-yield crossings.
- **Hard backstop:** absolute pop-cap (~2000) + wall-clock; on trip, **stamp `truncated` / `bridge-starved`** — never a silent stop. Halting **fails open** (surfaces low-corroboration strands with a stamp rather than hiding them).

## Resolved floor: forgetting

- Forgetting = **downward tier movement** `HOT→WARM→COLD→ARCHIVE-STUB`, **never deletion** while any provenance edge points at a strand. The archive stub (content hash + independent roots + timestamps) is immortal.
- **Decay sets pressure; gates set permission.** Eviction below COLD requires ALL of: low echo-discounted unique value vs *independent* neighbors; fresh independence stamp; not the outranked side of an active contradiction; not an earned bridge; temporally-discounted independent-source count ≤ 1; past a grace floor.
- New observed strands are pinned WARM for an un-forgeable grace window. Same-root floods collapse to multiplicity 1. A majority vote may only collapse *agreeing same-root echoes* — it may **never** pick a winner among disagreeing independent roots.

## The hard theorem (why the external layer is mandatory)

A multi-agent adversarial council proved that **claim adjudication cannot be solved from inside the graph**:

> Under "identity is priced, not prevented" (a patient attacker can pay a finite cost to mint independent-looking sources), there is **no purely internal rule** that both (a) lets one true witness overturn a planted false canonical AND (b) stops two fake sources overturning a true incumbent. The two requirements are in direct tension.

Three fatal attacks with no *internal* repair:
1. **Contradiction-set bomb** — drip in 500 individually-plausible observed claims about one attribute; each is individually un-prunable and none reaches quorum, so the answer drowns in honest-looking noise the system is forbidden to forget. Denial-of-answer at linear cost.
2. **First-arrival trap** — any incumbent-protecting threshold protects whoever arrived first, and the attacker controls arrival order.
3. **No second independent lock** — every candidate (path-disjointness, age axis) was either correlated with the first lock or anti-correlated with the threat.

The web can judge **what** a fact is; it cannot judge **who** is really behind it. Independence is a property of identity, and identity is forgeable from inside the graph. Therefore identity must be witnessed from **outside** — the Source-Identity Layer below.

## Source-Identity Layer (designed)

Mental model: **passport control at the border of the memory.** Before any source counts as a distinct witness, it shows ID at the door. The web stops *guessing* whether two sources are independent and *asks this layer*, which roots the answer in real-world-scarce things that are expensive to fake. Four pillars:

1. **Passport — one stable source id per source.** Proves *sameness*: the registry hands every source ONE deterministic id (a plain checksum of issuer + subject), so same id = same source. Lets the web collapse echoes (two facts from one source id are never corroboration). Cheap to mint, so necessary but **not sufficient** for independence. (Phase 2: this is a plain registry entry, not a cryptographic key — the process consuming a configured IdP/registry is the trust boundary.)
2. **Anchor — independence measured against scarce external roots, never declared.** Each key binds to one or more **anchors** (domain, verified human, org, device attestation, posted stake) — things costly/rate-limited in the real world. Independence between two sources = **disjointness of their anchor sets**, weighted by anchor cost. Same anchor → echo; different *costly* anchors → genuinely independent. This is "identity is priced, not prevented" made mechanical: the web is handed the *price* and weighs cheap identities as cheap.
3. **Credit score — reputation, earned slowly, lost fast.** Weight comes from track record (past claims later ratified by independent anchors vs contradicted), not headcount. Kills the **contradiction-bomb** (500 fresh sources start near-zero, loud but weightless) and the **first-arrival trap** (canonical follows earned trust, not arrival order).
4. **Attribution — permanent named authorship makes lying cost something.** (STAKING IS RETIRED as a feature; attribution replaces stake.) Every fact is permanently attributed to a named source, and the "disown a bad source" retroactive sweep claws back reputation across everything that source ever asserted — being caught wrong is expensive in earned trust, and the un-erasable attribution IS the deterrent. The FINANCIAL_STAKE anchor row survives as inert table data only (no producer, no ledger).

### Interface to the web

For every incoming observed fact the layer emits a stamp: `{ source_id, anchor_set, anchor_cost, reputation, stake_posted }` (`stake_posted` is a frozen shape field, constant 0 — staking retired). The two quantities the web **cannot compute about itself** — `provenance_independence` (the per-edge halting weight) and the "independent root count" used by the forgetting layer — **stop being self-computed and are read from this stamp instead.** Everything else in the web (activation, traversal, demotion, consolidation) is unchanged.

This honors both invariants: identity comes from outside (web is not its own identity witness), and the layer is **mechanical** — ids, anchors, scores — so the model still never witnesses.

### Anchor-cost table (product decision — tune to threat model)

`independence_weight` is what one anchor of this class contributes to the independence score between two sources whose anchor sets are disjoint. `rep_cap` is the maximum reputation a source rooted *only* in this class may ever earn (a ceiling, so a cheap anchor can never climb to high trust no matter how long it behaves). Costs are illustrative starting points, not fixed.

| Anchor class | What it proves | Cost to mint another | independence_weight | rep_cap | Notes |
|---|---|---|---|---|---|
| **None (bare key)** | sameness only | ~free | 0.00 | 0.05 | Echo-collapse only; carries no independence. Default for anonymous input. |
| **Email / OAuth account** | a controlled inbox | ~free, rate-limited | 0.10 | 0.30 | Disposable domains weighted to ~0; major-provider OAuth higher. |
| **Phone / SIM** | a provisioned number | low ($), rate-limited | 0.20 | 0.40 | VOIP ranges discounted; carrier-verified higher. |
| **Domain (DNS + TLS)** | control of a registered name | $/yr, registrar-throttled | 0.35 | 0.60 | Age + independent inbound links raise effective weight. |
| **Hardware attestation** | a distinct real device | device $ + enclave | 0.45 | 0.65 | TPM/secure-enclave signature; resists pure-software Sybil. |
| **Verified human (KYC)** | a distinct legal person | high, identity-checked | 0.70 | 0.90 | One human ≈ one anchor; collusion still possible but not free. |
| **Organization / institution** | an accountable entity | high, slow | 0.75 | 0.92 | Reputation-bearing; misbehavior has out-of-band consequences. |
| **Financial stake / bond** | skin in the game | scales with $ locked | 0.30–0.85 (∝ stake) | 0.85 | **RETIRED as a feature** — inert data row; no producer. Attribution replaces stake. |
| **External authority / notarized event** | a real-world fact vouched for by a trusted third party | highest | 0.90 | 0.98 | The window in "wall-with-a-window": ratifies derived → observed. |
| **Owner** | the deployment owner's own word | n/a — the PERSONAL preset's trust root | 0.90 | 0.98 | External-authority grade; auto-provisioned by `createAgentMemory()`. |
| **System of record** | a registry-configured authoritative system | deployment config, slow | 0.90 | 0.98 | Workday-for-HR etc.; asserted by the ENTERPRISE registry config. |
| **Local document** | an owner-admitted file/document | ~free, owner-gated | 0.35 | 0.60 | Domain-grade weight; the owner's admission is the gate. |
| **SSO tenant member** | membership in an IdP tenant | near-free (a fresh tenant is cheap) | 0.12 | 0.30 | DELIBERATELY email-grade; a config-verified custom domain adds the DOMAIN claim. |
| **Publisher (unverified)** | fetched web content, no track record | ~free | 0.04 | 0.10 | Collapses to eTLD+1; near-noise weight. |
| **Publisher (tracked)** | a publisher with configured, earned tenure | config + tenure | 0.18 | 0.35 | Kept BELOW Domain's 0.60 rep_cap deliberately. |

**Rules for the table:**
- **Independence is set-disjointness, not addition.** Two sources sharing *any* anchor are not independent on that anchor; their independence is driven by their *non-overlapping* costly anchors. Don't let a source self-stack ten cheap anchors to fake one expensive one — independence between two sources caps at the cost of their cheapest *shared* root being absent, and a single source's total is sublinear in its own anchor count.
- **rep_cap binds the ceiling; reputation still has to be earned up to it.** A KYC source is not born trusted — it can merely *eventually* reach 0.90.
- **Staking is retired; attribution replaces stake.** The stake-composition math survives only as inert code on an inert row; permanent named attribution plus the disown clawback is the deterrent.
- The whole table is **one swappable trust root** (like a Certificate Authority set). Changing which anchor classes you accept and their weights *is* the security policy. This is the intended knob — not a wall to break.

### Residual assumption (state it in any threat model)

This does **not** make Sybil impossible — it converts an unbounded, free attack into a priced, visible, self-limiting one. The anchors themselves are an external trust root (the deployment owner, the company's IdP/SSO, the configured source-of-truth registry, DNS ownership), exactly where TLS and CAs bottom out. The achievement is **shrinking all trust assumptions down to this one small, well-understood, swappable root** instead of scattering them through the graph.

## Strand data model (council-converged fields)

Any implementation of a strand should carry at least:
- activation register (per-traversal) + `refractory_until` timestamp
- incremental ancestor-sketch (MinHash/HLL over offline-assigned independence-class ids) → `convergence_factor`, **for ordering only**
- edge weights `w = link_confidence * provenance_independence * recency` + `out_weight_sum`
- `edge_type` with explicit `CROSS-WEB-BRIDGE` marker (typed structural class)
- `earned_bridge_value` + `far_side_potential` (accrue only from crossings ratified by ≥2 independent roots; recomputed offline — the query stream cannot write them)
- bridge sub-budget counter + low-yield-crossing counter
- provenance root-set with per-root independence-class + establishment timestamp
- salience `s` with `last_fire_time`, decay `λ`, real-retrieval `fire_count`
- `description_value` (reconstruction-loss bits vs independent neighbors; echo-discounted) — consolidation-eligibility gate only
- `fact_state` enum + observed/derived flag + the single outranking edge that explains a demotion
- observation-age (grace floor) + external-re-observation-count
- contradiction-set membership id + per-(entity,attribute) co-equal-claim cardinality
- `content_hash` + cold-store backpointer; `tier` enum; a reason stamped on every stop/tier-move (in code: `HaltStamp.reason` for stops, `Strand.last_tier_reason` for tier moves — `src/core/types.ts`)

## Design philosophy for future work

- Prefer **structural** defenses (a property an attack mathematically can't satisfy) over **policy** defenses (a threshold an attacker can tune around). Share-normalization beating hub spam is the template.
- When tempted to let the model decide something load-bearing (when to stop, what's canonical, what's independent), **don't** — route it to a gate, a provenance check, or an external signal. The model proposes; it never witnesses.
- Treat every new keep/prune/promote rule as adversary-facing: ask "what does a patient attacker who can pay to mint identities do to this?" before trusting it.
