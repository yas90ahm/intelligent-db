# Intelligent DB — Target Architecture

> Produced by an architecture council (4 specialist architects + a Sybil/gaming red-teamer + a chief architect) in response to four feedback points: Sybil detection, retroactive-undo depth, the trust-scoring algorithm, and the tamper-evidence guarantee. This is the **target**; the "Enforceable Today vs Needs Building" table is the honest split against the current prototype (~138 passing tests, SQLite-persistent, crash-consistent). See `CLAUDE.md` for what is implemented.

## One-paragraph architecture

Intelligent DB is a trust-aware, append-only memory for an AI agent swarm built on four interlocking pillars. **Identity** is cryptographic: every source is an Ed25519 passport whose `source_id = sha256(pubkey)`, bound (in the target) to scarce real-world *anchors* (domain, email/phone, hardware, KYC, financial stake) through a pluggable, attestation-emitting binding pipeline; independence between sources is computed as an exact maximum-independent-set over anchor-disjointness, gated by a sublinearly-capped anchor-cost table, so the *price* of N independent identities — never their impossibility — is the security parameter. **Provenance** is a DAG: derived strands cite the exact strands they were computed from, so `disown(source)` runs a BFS taint-closure that demotes (never deletes) all reachable derivations and a corroboration-event ledger reverses the exact reputation those strands earned, class-bounded so coincidental independent agreement is spared. **Trust** is a per-source reputation — moving from today's ad-hoc multiplicative update to a Beta(α,β) model with independence-weighted pseudocounts, asymmetric (contradiction weighs 4×), exponentially decayed, and read out as a lower-confidence bound capped at the anchor-derived `rep_cap` — that drives a decisive-or-defer adjudicator which auto-resolves disputes only on a clear earned margin and otherwise escalates to a human. **Integrity** is an append-only, hash-chained, Ed25519-signed ledger persisted to SQLite, hardened (in the target) with a Merkle tree, epoch Signed Tree Heads, and externally-published/witnessed roots for O(log n) inclusion and consistency proofs. The honest through-line: every guarantee is *evidential and priced*, conditional on an honest latest-hash anchor, uncompromised signer keys, sound anchor binding, and correct offline class assignment — the system is designed to **fail closed** (no anchor ⇒ no independence ⇒ defer) rather than to be unbreakable.

## Per-pillar precise guarantee

**Sybil / Identity.** For an attacker who has spent real-world cost C on anchors, the maximum number of mutually-independent identities presentable to any adjudication is bounded by the maximum-independent-set over anchor-disjointness, where each node's admissibility requires anchors whose summed, sublinearly-capped cost is affordable under C. *Independence is priced, never prevented.* Holds conditional on: (a) anchor-binding soundness — a passed proof genuinely evidences control of a scarce root; (b) an honest signed-latest-hash ledger anchor; (c) unbroken Ed25519/SHA-256; (d) correct offline independence-class assignment. Hard edge: a funded attacker who *buys* N genuinely-distinct anchors gets N real independent identities, and the cost table must price *fleet size* per registrar/ASN/issuer or cheap-anchor fleets defeat the bound.

**Undo / Provenance.** Given a complete DERIVATION graph and a total corroboration-event ledger, `disown(source)` deterministically and idempotently (1) demotes every strand transitively derived from the disowned source's strands and (2) reverses the *exact recorded* reputation those tainted strands caused on each beneficiary, bounded by independence class. Append-only: demoted strands persist as immortal archive stubs. Holds *only* for influence that is both graph-reachable via DERIVATION edges and recorded as a corroboration event. The undo is **epistemic, not causal**: it corrects beliefs and credit, not actions already taken in the world, nor uncited/laundered influence.

**Trust-scoring.** A source's weight is a monotone function of anchor-*independent* evidence mass for and against it, capped at `rep_cap`, recoverable only at a bounded rate: gain per corroboration is strictly sublinear in headcount (k colluders in one anchor class count ≈ once), and recovery cost from a contradiction strictly exceeds the gain it erased. Low-evidence sources are penalized for *uncertainty* (lower-confidence-bound readout), so whitewashing a fresh identity buys near-zero weight. Assumes correct independence-class/anchor-cost assignment upstream — the Beta layer faithfully amplifies whatever independence accounting it is fed, and *no more*.

**Tamper-evidence.** Given (A) a verifier holding an authentic, out-of-band copy of the latest record's hash (the anchor) and (B) uncompromised honest signer keys, any mutation, reordering, or post-anchor truncation of records at/before the anchored seq is *detected* by `verifyChain()`, which returns the first broken seq; with Merkle + published/witnessed STHs, whole-log deletion, rollback, and split-view equivocation are also detected. This is **detection, not prevention**, and is null without a trusted anchor — anyone holding *a* valid signer key can re-sign a coherent forged chain from genesis. A compromised signer key (authoring valid malicious records) and availability/destruction attacks are out of scope for this pillar.

|---|---|---|

## Key design decisions

**1. Anchor-binding pipeline — BUILD, pluggable, fail-closed.** A `AnchorBinder` interface yields signed, expiring `AnchorAttestation{ sourceId, anchorType, anchorId (salted hash of the root — never raw PII), proofRef, weight, classId, notBefore, notAfter, verifierSig }` appended to the ledger. Concrete binders: **DOMAIN** (DNS-01 TXT challenge or HTTP-01; `classId = eTLD+1`), **EMAIL/PHONE** (signed-nonce round-trip; classId = normalized address/E.164), **HARDWARE** (TPM 2.0 / WebAuthn; classId = EK/AAGUID chain), **KYC/VERIFIED_HUMAN** (OIDC4VC verifiable credential; classId = issuer-scoped pseudonym), **FINANCIAL_STAKE** (time-locked deposit proof; weight 0.30→0.85 by size×duration). The cost table alone prices a *single* anchor but not a *fleet*; therefore add aggregate-independence caps keyed on deterministic *operator* classIds — WHOIS registrar + hosting ASN for domains, issuer for KYC — so 10k cheap domains behind one registrar collapse toward one class rather than 10k. High `rep_cap` requires ≥2 disjoint anchor types proven to bind the *same* pseudonymous subject (reinforcing one identity), not one human spreading anchors across many identities. Attestations expire and must re-bind, imposing ongoing cost. Absent any binder a source stays BARE_KEY weight 0 — writable and able to earn corroboration, but contributing 0 independence, so its disputes defer to a human.

**2. Trust algorithm — ADOPT Beta(α,β) + decay; retain multiplicative *asymmetry as the update shape*.** Counts never drive weight, independence does. State `{α, β, lastUpdate}`, prior `Beta(1,1)`. Corroboration: `α += w`, `w = independenceWeight ∈ (0,1]` taken from the MIS result (k colluders in one class add `w` once, total — this is where headcount is denied). Contradiction: `β += c·w`, **c = 4**. Decay on each access: `α ← 1 + (α−1)·λ^Δt`, same for β, **half-life ≈ 90 days** (`λ = 0.5^(1/90d)`). Weight = `min(rep_cap, mean − z·sd)` of the Beta, **z ≈ 1.0** (lower-confidence bound). `reverseCredit` stays exact-ledger (subtract recorded `w` from α). The LCB alone must *never* clear a decisive margin for high-impact/irreversible decisions — those additionally require a minimum corroboration count, a recency-clean window, and a winner whose independence derives from ≥2 disjoint anchor classes; rate-limit contradictions per source-pair to block forced-review griefing.

**3. Merkle tree + published root — YES.** Keep the chain (total order, cheap), add a Merkle tree (RFC 6962: leaf = `sha256(0x00‖preimage)`, node = `sha256(0x01‖L‖R)`) for O(log n) inclusion *and* consistency proofs. Emit an epoch **Signed Tree Head** `Ed25519(sha256(tree_size‖root‖timestamp))`, and **publish each STH to ≥2 independent append-only sinks** the operator cannot rewrite. This upgrades "tamper-evident given a trusted anchor I hold" to "tamper-evident against the server operator itself": deletion, rollback, and split-view equivocation become non-repudiable (two conflicting STHs over the same `tree_size`). Publish the **genesis STH at deployment**.

**4. Undo-engine hardening.** (a) Add **weak-influence edges** (consulted-but-not-cited reads) that, on disown, emit a *review-queue* entry rather than auto-demoting. (b) Make the corroboration ledger **total**: assert at write-time that every ratify emits an event; add a reconciliation audit per source. (c) Record **adjudication provenance** (winning margin + contributing strand-ids); on taint, if removing tainted strands drops the margin below threshold, **re-open the dispute**. (d) Strands whose independent corroboration survives removal of the tainted set are not demoted, blocking false-disown-as-suppression. Keep demote-never-delete and taint-generation counters for O(1) idempotency.

## Threat model

**An attacker CANNOT (given the assumptions hold):** inflate a single key into many witnesses; flood fake-independence and have it counted (MIS → true count); stack cheap anchors into a costly one (sublinear cap); re-register the same root as a new class (deterministic classId); profit *durably* from corroboration headcount (independence-weighted α); whitewash a fresh identity into weight (uncertainty-penalizing LCB + anchor cost 0); poison memory and keep the laundered downstream conclusions or the reputation they earned (taint-closure + exact reverseCredit); silently edit, reorder, or post-anchor-truncate the ledger (verifyChain), or — once roots are published and witnessed — silently delete, roll back, or split-view the whole log.

**An attacker CAN (residual, honest):** buy N *genuinely distinct, genuinely independent* anchors and get N real independent identities (priced, not prevented). Beyond that: (i) **patient single betrayal** — farm genuine corroboration, saturate α, then defect once; decay + 4×-contradiction stop *repeat* offense but a high-value first poison can clear adjudication before a contradiction arrives — mitigated by gating irreversible decisions on count + recency. (ii) **Cross-anchor collusion** — one human/broker controlling multiple "independent" anchor issuers; mitigated by common-subject binding + issuer denylists, but a clean colluding KYC vendor defeats it. (iii) **Compromised signer key** — authors valid malicious records; requires key custody/rotation, orthogonal to tamper-evidence. (iv) **Availability/destruction** — deleting the live DB destroys service even when detectable. (v) **Laundered influence** — uncited reasoning and real-world actions already executed are not undoable. (vi) **Offline class-assignment error** — a human who fails to place correlated roots in one class silently weakens every downstream bound.

## Roadmap (by leverage)

1. **Anchor-binding pipeline (highest leverage).** Today every source is BARE_KEY independence 0, so the entire Sybil/trust apparatus is structurally real but operationally inert. Ship DOMAIN (DNS-01) and EMAIL/PHONE binders first, each emitting signed expiring attestations with deterministic classIds, **including the per-registrar/ASN/issuer fleet-size caps from the start** (a binder without fleet-pricing manufactures false independence).
2. **Tamper-evidence external anchor.** Merkle + epoch STH + publication to ≥2 sinks; **publish the genesis STH at deployment.** Removes "the operator can `DELETE FROM ledger`"; the gap is undetectable until publication is live.
3. **Beta(α,β) + decay trust scoring.** Swap the ad-hoc multiplicative update for the calibrated Beta/LCB/decay model. Depends on (1) for `w` to be meaningful. Ship with the high-impact-decision gate.
4. **Undo-engine hardening.** Total-ledger invariant + reconciliation, then adjudication-provenance + dispute re-opening, then weak-influence review-queue edges.

**Residual risk after the full roadmap, stated plainly:** the system will be tamper-evident (not -proof), Sybil-priced (not -prevented), and epistemically reversible (not causally reversible). Its strength is exactly the strength of the anchor-binding pipeline's soundness, the honesty of the published-root witnesses, the custody of signer keys, and the correctness of offline independence-class assignment. None of those four is solved by code in this repo; three are operational/social, and one (class assignment) is a standing human-judgment liability. A funded, patient, anchor-buying adversary remains in scope by design.
